# Sessions — Per-Session Summaries

One file per session, written by the `session-handoff` skill at wrap-up.

## Structure

Files are named `YYYY-MM-DD-<topic-slug>.md`. The skill auto-generates these on session end.

## When to use

- Catching up on a specific topic from prior sessions
- Understanding why a decision was made
- Recovering context when resuming a thread

## When NOT to use

- Live working notes during a session → `memory/staging/`
- Long-term decisions or patterns → `memory/historical-decisions.md`
- Reviews of recurring patterns → `memory/reviews/`
